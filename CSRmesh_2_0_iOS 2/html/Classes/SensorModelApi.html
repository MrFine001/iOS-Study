– getTypes:firstType:success:failure:

getTypes - request the Sensor Types that follow from the given sensor. Upon a successful transaction the success code block is executed with the parameters (NSNumber deviceId, NSArray sensorTypes) and the callback didGetSensorTypes is invoked. Upon a failed transaction the failure code block is executed and the callback MeshServiceApi:didTimeoutMessage is invoked.

– setValue:sensors:acknowledge:success:failure:

setValue - Write the given value to the given sensor. Up to two sensors can be written in one command. If success is not nil then the meshRequestId will be returned, otherwise nil will be returned. If success code block is not nil then it will be executed upon a successful transaction and the callback didGetSensorValue will be invoked. If the transaction fails then the failure code block is executed. If the success parameter is nil then CSRmesh acknowledgement will be supressed.

- (NSNumber *)setValue:(NSNumber *)deviceId sensors:(NSDictionary *)sensors acknowledge:(NSNumber *)ack success:(void ( ^ ) ( NSNumber *deviceId , NSDictionary *sensors ))success failure:(void ( ^ ) ( NSError *error ))failure

Parameters

deviceId
  • (NSNumber *) The ID of this device. Refer to the delegate MeshServiceApi.didAssociateDevice
sensors
  • (NSDictionary) - The key is the NSNumber of the sensor Type as an unsigned short and the Value is the NSNumber of the sensor value of size one to four octets. Temperature value should be in Kelvin and of type float wrapped in an NSNumber.
success
  • Upon a successful transaction the success code block will be executed with the parameters (NSNumber deviceId, NSDictionary sensors)
failure
  • Block called upon error

Return Value

meshRequestId - (NSNumber *) The id of the request. Pair up with the id returned in didGetPowerState.

– getValue:sensorType1:sensorType2:success:failure:

getValue - Read the value for the given sensor(s). A read request for up to two sensors can be made in one command. Upon a successful transaction the success code block will be executed and the callback didGetSensorValue will be invoked. If the transaction is unsuccessful the failure code block will be executed and the callback MeshServiceApi:didTimeoutMessage will be invoked.

- (NSNumber *)getValue:(NSNumber *)deviceId sensorType1:(NSNumber *)sensorType1 sensorType2:(NSNumber *)sensorType2 success:(void ( ^ ) ( NSNumber *deviceId , NSDictionary *sensors ))success failure:(void ( ^ ) ( NSError *error ))failure

Parameters

deviceId
  • (NSNumber *) The ID of this device. Refer to the delegate MeshServiceApi.didAssociateDevice
sensorType1
  • (NSNumber *) The first sensor type as an unsigned short wrapped up in an NSNumber.
sensorType2
  • (NSNumber *) The second sensor type as an unsigned short wrapped up in an NSNumber. This is optional and may be nil.
success
  • Code block executed upon a successful transaction. The success code block parameters are (NSNumber deviceId, NSDictionary sensors)
failure
  • Code block executed upon an unsuccessful transaction.

Return Value

meshRequestId - (NSNumber *) The id of the request. Pair up with the id returned in didGetPowerState.
