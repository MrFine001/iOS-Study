MeshServiceApiDelegate Protocol Reference

Conforms to NSObject
Declared in MeshServiceApi.h

Overview

@header MeshServiceApi Class is a part of the CSRmesh Api and provides a set of Api methods to Initialise the CSRmesh library, Discover and Associate Mesh Devices, Control a mesh Device. The Api also provides support for REST calls and therefore the frameowrk CSRmeshRestClient.framework should be included in the project.

– didUpdateAppearance:appearanceValue:shortName:

didUpdateAppearance delegate. Invoked by the CSRmesh library.

- (void)didUpdateAppearance:(NSData *)deviceHash appearanceValue:(NSNumber *)appearanceValue shortName:(NSData *)shortName

Parameters

deviceHash
  • NSData. This is the lowest 31-bits of the SHA-256 of the DeviceUUID of New Device.
appearanceValue
  • NSNumber. The Appearance field is a 24-bit value that is composed of an “organization” and an “organization appearance”. The following organizations are defined: 0x00 = Bluetooth SIG For the Bluetooth SIG organization, the organization appearance is from the assigned numbers for the Appearance Characteristic Value.
shortName
  • NSData. The ShortName field is the first 9-octets of the name of this device. This is set by the manufacturer of the device.

– didTimeoutMessage:

didTimeoutMessage delegate. Invoked by the CSRmesh library to indicate loss of acknowledgement. Each of the acknowledged Api methods when called returns a meshRequestId. The same meshRequestId is used in this callback to indicate a loss of acknowledgement for the given messageId within the given timeout. The timeout can be computed as the (number_of_retries x retry_interval) (Deprecated: Replaced with failure code block.)

- (void)didTimeoutMessage:(NSNumber *)meshRequestId

Parameters

meshRequestId
  • NSNumber. This is an int32 and is the ID of method call made into the library.
